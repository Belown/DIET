# Plan: Chapter 2 — Interactive Fiction UI

## Goal

Wrap the existing Chapter 2 investigation mechanics in an interactive fiction-style UI shell. Keep all game logic untouched — only change how the player experiences it: text-driven narrative with typewriter reveals, clickable choices, and seamless flow instead of the current 4-tab switcher.

---

## What changes

| Before | After |
|--------|-------|
| 4 pill tabs to jump between sections | Linear text-based narrative that flows naturally |
| Static text blocks | Typewriter text reveal, paragraph by paragraph |
| Sliders/checkboxes directly in panels | IF-style choices that open into the existing interaction panels |
| "Continue →" buttons scattered | Choices presented as numbered options at the end of each text passage |
| Results shown as raw accuracy numbers | Results embedded in narrative text ("The model predicts a 72% chance you're a threat...") |

## What stays the same

- All game state (`currentDay`, `dayPlans`, `dayLocked`, budget system, `summarizeStrategy`)
- The scatter plot with draggable boundary
- Zone selection cards, population sliders, question chips
- Accuracy computation (`regionAccs`, `otherCityAccs`, `overallAcc`)
- The detective image and existing CSS (adapted to DESIGN.md)

---

## Implementation (3 steps)

### Step 1: Build the IF UI shell components

**`NarrativeBox.tsx`** — the core IF display panel:
- Shows the current story passage as text
- Typewriter reveal: text appears character-by-character with a blinking cursor
- Click-to-skip: clicking anywhere completes the current paragraph instantly
- Styled as a bordered card with `var(--rui-surface)` background, per DESIGN.md

**`ChoiceList.tsx`** — renders a list of clickable choices below the narrative text:
- Each choice is a pill button (`--radius-pill`, weight 500, no shadow)
- Choices appear with staggered `animation-delay` (50ms increments)
- On select, triggers the next narrative passage or opens an interaction panel

**`NarrativeBox.module.css`** — styling:
- Dark text (`var(--rui-dark)`) on light surface (`var(--rui-surface)`)
- Generous padding (32px), card border radius (`--radius-card`)
- Typewriter cursor: a 2px-wide `var(--rui-dark)` bar with opacity pulse animation
- Zero shadows per DESIGN.md

**`ChoiceList.module.css`** — styling:
- Pill buttons: 9999px radius, 12px 24px padding, `var(--rui-dark)` bg on hover
- Transition: `background 0.2s ease, color 0.2s ease`
- Stacked vertically with 10px gap

### Step 2: Rewrite Chapter2SamplingBias.tsx to use the IF shell

Replace the current `step` state machine ("story" | "demo" | "investigate" | "results") with a **passage-based** system:

```typescript
type PassageId =
  | "intro-1" | "intro-2" | "intro-3"       // existing story lines as passages
  | "demo-intro" | "demo-draw" | "demo-reveal" | "demo-failure"
  | "day1-brief" | "day1-plan" | "day1-debrief"
  | "day2-brief" | "day2-plan" | "day2-debrief"
  | "day3-brief" | "day3-plan" | "day3-debrief"
  | "verdict" | "epilogue";
```

Each passage has:
- `text: string` — the narrative text for this passage
- `choices?: { label: string; nextPassage: PassageId }[]` — what to show after text reveals
- `panel?: "scatter" | "investigation" | "results"` — if set, render the gameplay panel below the text

Flow:
1. Player sees narrative text with typewriter effect
2. When text completes (or is skipped), choices or a panel appear
3. Choosing advances to the next passage
4. Investigation panels (zone picker, sliders, etc.) appear at `dayN-plan` passages
5. Results panels appear at `verdict` passage

### Step 3: Adapt investigation panels to fit inside the narrative box

The existing zone cards, sliders, and question chips remain as functional components but:
- They render *inside* the narrative panel (below the passage text, before choices)
- They keep their current layout and functionality
- A "Confirm" choice appears below them that triggers `sendDetective` and advances to the next passage

## DESIGN.md compliance notes

- Zero shadows
- All fonts weight 500 (display) or 400 (body)
- Pill buttons at 9999px radius
- `var(--rui-dark)` for primary text/elements, `var(--rui-surface)` for panels
- Positive letter-spacing on body text (0.16px)
- Near-black + white palette for marketing/UI surfaces

## Out of scope

- New characters (Detective Vale, judge)
- Branching narrative with hidden variables
- Multiple endings beyond what the accuracy numbers already show
- `localStorage` save/load
- Engine extraction for other chapters
